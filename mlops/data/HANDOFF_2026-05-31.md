### 2.2 루틴 생성 rest_seconds 프롬프트 가이드 추가

`rag.py` `_build_routine_prompt()` Rules 섹션에 목표별·운동 유형별 가이드 추가.

**문제**: LLM 프롬프트에 `rest_seconds` 기준이 없어 모든 운동에 동일한 값 출력.

**해결**: reps 가이드와 동일한 패턴으로 추가
```
- strength compound (squat/deadlift/bench): 180-300s
- strength accessory (curl/row/isolation): 60-90s
- hypertrophy: 60-90s / endurance: 30-60s
- rehabilitation: 60-120s / weight_loss: 30-45s
```

추가로 `routine_targets.py`에 `rest_seconds` 상한 클램프(`min(600, ...)`) 적용.

**PR**: #212 — 머지 완료 ✅

## 7. 알아두면 좋은 컨텍스트

### muscle_groups.name 혼재 문제
DB `muscle_groups.name`이 PascalCase(`Front Deltoid`)와 영문 slug(`anterior_deltoid`) 혼재.
Gemini는 소문자 해부학 표준명 반환 → `_SLUG_ALIASES`로 대응 (PR #213).
새 운동 추가 시 alias 목록 업데이트 필요할 수 있음.

### 루틴 생성 구조
```
논문(ChromaDB) → 운동 선택 이유 제공 (justification)
헬스장 기구 목록 → 실제 운동 선택의 원천
LLM → sets/reps/rest_seconds 제안 (가이드 기반)
백엔드 → weight_kg만 1RM 계산으로 결정
```

### rest_seconds 동작
- LLM이 프롬프트 가이드 기반으로 운동별 값 제안
- `routine_targets.py`에서 `_coerce_int(llm_rest_seconds, DEFAULT_REST_SECONDS=90)` 우선 적용
- 상한: `min(600, ...)` / 하한: `max(0, ...)`
